Tittlen på artikkelen

Git og tjenester som Github er en veldig viktig del av utviklingsverden. Ved hjelp av Git kan man holde styr på hva man skal gjøre i et prosjekt, samtidig alle endrigene som blir gjort på prosjektet. Det finnes mange ulike måter å bruke Git og Github på, prosjekter som forskjellige mennesker jobber på over hele verden er nok noe av det mest effektive. Ved hjelp av Github, kan utviklerer jobbe sammen uansett hvor de er i verden, de kan også få ideer av hva som bør oppgraderes og gjøres, ved hjelp av tips fra brukere av hva som kan bli bedre. På denne måten vil utviklere alltid være oppdatert på hva som bør gjøres, hva som skal gjøres og hva som kan bli bedre ute i hverdagen, på en rask og ryddig måte.

Git er en rask måte å jobbe på. Med Git vil alle operasjoner bli gjort lokalt, istedenfor å bruke tid til å konstant kommunisere med en server. Takket være dette, vil Git bruke mye mindre tid til å jobbe og lagre prosjekter, enn det mange andre tjenester gjør.

Teknologien gjør det også mye mindre katastrofalt å få en feil i prosjektet. Git lar deg laste opp en tidligere versjon av prosjektet, i en situasjon hvor du tidligere måtte bruke tid til å feilsøke, kan du bare laste opp en tidligere versjon uten feilen, og så rette opp feilen senere i filen som inneholderden. Når det kommer til gjennopretting, har du det ordentlig trykt på Git. Git's brukere har en full backup i tilfelle corrupte filer eller crash. Hver og en av disse filene kan byttes med den orginale filen, slik at prosjektet ikke blir blokkert av uheldige omstendigheter.

Dette synes jeg om git/github etter å ha brukt det.
